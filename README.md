# Тестовое задание

Приложение выполнено на ASP.NET Core Web API в связке с Angular.  
Web API реализован по принципам Onion Architecture: все слои взаимодействуют друг с другом строго через интерфейсы, определенные в нижележащих слоях.  
  
## Архитектура

Проекты Domain уровня содержат сущности, пользовательские исключения (Domain.Entities), а также интерфейсы репозитория (Domain.Contracts).  

Проект Repository реализует эти интерфейсы и также отвечает за контекст базы данных.  

Проекты Service уровня содержат абстракции, инкапсулирующие бизнес логику (Service.Contracts), и их реализацию (Service).  

Проект Presentation содержит отделенные от основного приложения (OrdersApp) контроллеры, а также класс для валидации моделей. Отделение контроллеров в отдельный проект позволяет предотвратить внедрение в них чего-либо кроме интерфейсов Service уровня.   

Проект WebUI содержит Angular приложение. Реализована главная страница с фильтрацией и выводом заказов.  

## Валидация моделей
Реализована через пользовательские атрибуты для Create и Put запросов.  

## Global error handling
Обработка исключений реализована с помощью встроенного middleware c детализацией кодов состояния ответов.  

## Логирование
Подключено с помощью NLog.  

## Фильтрация
Реализована через параметры в строке запроса. Помимо обозначенных в задании фильтров, через query string также реализована pagination на серверной и клиентской части.
